# How I picked my first programming language

I was 14. My older cousin had a copy of "Learn Python the Hard Way" on his desk, and I borrowed it for a weekend that turned into a year. The book was awful, honestly. It made you retype everything. But by the time I'd finished it, I'd built two small games and a script that scraped my school's lunch menu and emailed it to me.

That last one is what hooked me. I'd never made a computer do something useful for me before. Print "hello" was fine. But getting the lunch menu by 8am, automatic, every weekday, that felt like cheating, in a good way.

Looking back, I don't think Python was special. Ruby would have done it. So would JavaScript, or Perl, or anything else with a decent string library and an SMTP module. The thing that mattered was the size of the wins. Small enough that I could finish them on a Saturday. Useful enough that I'd brag about them at lunch on Monday.

Some advice for anyone starting out: don't pick the language with the cleanest tutorials. Pick the one your friends use. They'll answer your questions when you're stuck at midnight, and that's worth more than any book.
